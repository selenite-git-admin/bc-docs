# Database naming convention — _dev/_prod suffix for environment safety

## Context

Explicit environment naming prevents cross-environment accidents. Cost is minimal (one-time config change), safety benefit is high.

## Decision

Renamed databases: bc_platform → bc_platform_dev, tbc_selenite → tbc_selenite_dev. Convention: bc_platform_{env}, tbc_{slug}_{env}. Prevents accidentally running dev scripts against production. ISO 27001 A.12.1.4 environment separation.
