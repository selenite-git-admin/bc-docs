# Contract Creation Framework — SC + AC Entry Boundary Pattern

## Context

SC + AC are the most frequently created contract types (~110 each per system). Standardizing the creation pattern with auto-generated envelopes, meta-schema validation, and governance state machine ensures consistency. AC requiring human configuration (not auto-generation) is deliberate — admission rules are business decisions, not derivable from catalog structure alone. The standalone AC wizard complements the SC wizard's 'also create AC' toggle for cases requiring custom DQC rules.

## Decision

Source and Admission Contracts are the entry boundary of the Source Chain. SC declares field inventory from source catalog. AC declares admissibility rules (DQC Bronze layer). AC depends on SC — cannot be created without an existing active SC. Both use 4-step wizard pattern in bc-admin with auto-generated barecount/{family}/v1 envelopes. SC can be generated programmatically via Contract Constructor; AC requires human configuration (rules, thresholds, validation policy). SOP governs both manual and programmatic creation tracks.
