# MC Body Purity — Catalog Fields Out, Formula as Object, sql_logic Parked

## Context

Evaluator needs computation fields only. Identity and classification are catalog concerns (queryable columns, not buried in JSON). pack_code/module_code replaced by 4-level taxonomy (DEC-7bbdba). Formula as object allows future extensibility (notation, latex). unit stays per original requirements — Metric Snapshot inherits it.

## Decision

MC body carries computation + output only. Refinements from master shape review:

1. **metric_code, display_name** → catalog table columns (identity, not computation)
2. **kpi_id, catalog_code, category, tier** → catalog table columns (classification, not computation)
3. **pack_code, module_code** → dropped (legacy taxonomy, replaced by domain/subdomain in header)
4. **value_type, unit_of_measure** → catalog table columns (display metadata). But `unit` stays in body per requirements.
5. **formula** → object format `{ "text": "O1 = (I1 / I2) * 90" }` per requirements, not flat string
6. **direction** → renamed `direction_code` per ISO 11179 naming standard (D078)
7. **sql_logic** → parked. Not in requirements. Revisit during implementation — may be needed as machine-executable form.

MC v1 body: 9 fields — input_type, formula, variables[], co_bindings[], temporal_gate, unit, direction_code, thresholds[]. Pure computation + output classification.
