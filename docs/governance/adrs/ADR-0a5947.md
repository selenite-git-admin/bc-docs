# bc-admin Information Architecture — 5 navigation groups

## Context

Superseded by D068. Original 5-group IA replaced by v5 structure.

## Decision

bc-admin organizes into 5 navigation groups with clean separation of concerns:

**Registry** — what we know + how we govern it
- Source Catalog: 6-tier hierarchy (Provider → System → Version → Module → Object → Field) with inline integration (connectors/connections/readers shown per system, not as separate nav)
- Source Landscape: per-source reference indexes (SAP 245K tables, SFDC 800+ objects, etc.). Planning tool for onboarding. NOT part of Source Catalog.
- Contracts: all 7 contract types organized by chain position — Admission (extraction + source + admission, source-bound), Canonical (CO definitions, source-agnostic), Metric (KPI + intervention, CO-bound), AI (intelligence, CO + metric bound)

**Onboarding** — getting new sources/tenants live
- Readiness Dashboard: gap analysis per provider (Source Landscape − Source Catalog = work remaining). Shows readiness matrix, not raw counts.
- Discovery Scans: tenant-scoped metadata reconnaissance (D046)
- Standard Mappings: pre-built canonical mapping templates (platform-scoped, Phase 1 auto-match)
- Tenant Mappings: custom/extended bindings per tenant per source (Phase 2/3, authored by platform team)

**Operations** — runtime health + execution
- Reader Runs: admission execution log
- Boundaries: SO/CO/Metric/Action boundary health + throughput

**Platform** — platform governance
- Tenants: tenant lifecycle, configuration, onboarding status
- Users: accounts, roles, RBAC, permissions
- Tickets: drift alerts, incidents, change requests
- Infrastructure: services, databases, deployments, health
- Libraries: SBOM, OSS dependency tracking

**Data Generators** — synthetic test data
- SAP / SFDC / Opcenter / AVEVA instances

Key removals:
- "Core" group renamed to "Registry"
- Business Catalog and Metric Catalog removed from bc-admin (these are bc-portal concepts — see D051)
- "Integration" dissolved — connectors/connections/readers shown inline on source system detail pages
- Core Map demoted from nav item to dashboard action or Registry landing visualization
- "Source Registry" in Onboarding replaced by Readiness Dashboard (gap analysis, not raw counts)
