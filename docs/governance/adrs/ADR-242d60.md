# AC Body Purity — Remove readiness_predicate

## Context

readiness_predicate governs WHEN admission starts, not how records are validated. That is an execution orchestration concern belonging to the Reader, not a quality contract concern belonging to the AC. Same principle as D245 — contract bodies carry their domain concern only, no runtime leakage.

## Decision

Admission Contract body declares validation rules and quality gates only. One field removed during master shape review:

**`readiness_predicate`** — orchestration pre-condition ("don't start admission until source batch is complete"). This is a WHEN concern (Reader/execution scheduling), not a HOW-to-validate concern. Moved to Reader execution config.

**`max_observation_age` retained** — "reject records older than P30D" is a quality rule (stale data fails admission). That is AC's job.

AC v1 master shape: source_contract_version_id, field_rules[], record_rules[], admission_bands, outcome_thresholds, validation_policy, max_observation_age.
