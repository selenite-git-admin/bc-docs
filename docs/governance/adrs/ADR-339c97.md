# External standards provenance on Business Objects and Business Fields

## Context

Enterprise customers in banking/payments already use ISO 20022. Referencing governed vocabularies gives external authority, enables interoperability, and adds credibility. Finance and procurement are well-covered by existing standards. Manufacturing has ISA-95. HR and CRM are less standardized — BareCount becomes the authority there.

## Decision

Business Fields and Business Objects carry optional provenance columns (source_standard, standard_ref) referencing external governed vocabularies:

- ISO 20022 — financial services (CashEntry, InstdAmt, EndToEndId)
- UN/CEFACT Core Component Library — trade/supply chain (Invoice, Amount.Type)
- ISA-95 / IEC 62264 — manufacturing (Equipment, BatchOrder)
- ISO 7372 (UNTDED) — EDI trade data elements

Provenance is reference-only, not a binding constraint. Fields/objects that don't map to any external standard have NULL values. BareCount is the authority for domains where no external standard exists (HR, CRM, industry-specific).

This enables: SAP BSEG.WRBTR → BareCount instructed_amount → ISO 20022 InstdAmt (three-layer traceability).
