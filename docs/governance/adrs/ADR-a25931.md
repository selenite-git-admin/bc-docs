# JSONB-first for boundary payloads, typed tables immediately after via D210

## Context

Getting E2E working fast validates the full chain before investing in typed table infrastructure. But JSONB is NOT the end state — D162 Rule 1 forbids queryable data in JSONB. The typed table trigger on OC activation ensures every new contract gets real columns automatically.

## Decision

E2E testing uses JSONB envelope tables (observed_payload_json, canonical_payload_json, metric_value_json) in boundary schema. Immediately after E2E validation, D210 DDL generator auto-creates typed tables on OC activation. Three-layer architecture: RDS envelope (metadata), RDS typed tables (query path), S3 archive (raw JSONB backup).
