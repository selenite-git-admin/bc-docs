# SFDC Source Catalog — Salesforce native cloud grouping, CPQ deferred

## Context

Salesforce's own cloud boundaries are the natural grouping — Sales Cloud objects relate to each other, Service Cloud objects relate to each other. This is how Salesforce customers think about their org. CPQ is a managed package (not standard) and only available in orgs that purchased it — mixing it with standard objects creates false expectations.

## Decision

Salesforce source catalog uses native cloud grouping for modules (Sales Cloud, Service Cloud, Marketing Cloud, Platform), NOT business-process grouping from the entity catalog. CPQ objects (SBQQ__Quote__c, SBQQ__QuoteLine__c, SBQQ__Subscription__c) are deferred — managed package objects, customer-specific. 19 standard objects in Phase 1. Bulk API and CDC connectors added as separate task, not Phase 1.
