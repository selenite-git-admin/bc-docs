# Governance lifecycle uses 'in_review' not 'submitted'

## Context

'in_review' better describes the state (actively being reviewed) vs 'submitted' (passive, implies only the submitter acted). The code already uses 'in_review' across bc-admin and bc-core. Changing code would require DB migration and API contract changes for no semantic improvement.

## Decision

The contract governance lifecycle state is 'in_review', not 'submitted' as written in the bc-docs contract-registry.md spec. The code (bc-admin + bc-core) uses 'in_review' consistently. Update the spec to match the implementation: draft → in_review → approved → released → deprecated → retired.
