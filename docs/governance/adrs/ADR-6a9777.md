# Inline ownerJson retained — owner_assignment table deferred until RBAC owner queries needed

## Context

Same cost-benefit logic as D089h. Owner data is an annotation, not a queryable relationship yet.

## Decision

6 tables use inline ownerJson JSONB for owner references. D089 rule 5 recommends a shared owner_assignment table. Decision: retain inline JSONB for now.

Rationale: (1) ownerJson stores {primaryOwner, team} metadata — not a simple FK. (2) No current use case for "find all entities owned by person X" cross-entity querying. (3) Junction table would add JOINs to 6 entity queries. (4) Owner assignment is currently a UI annotation, not an RBAC enforcement mechanism.

Trigger for revisiting: when RBAC-based ownership queries or "my entities" dashboards are needed.
