# Frontend is React 18 + Vite, not Angular

## Context

Codebase exploration confirmed: apps/web uses React with Vite, not Angular. The existing frontend has well-defined TypeScript interfaces in src/types/ and React Query hooks in src/hooks/ that the NestJS backend must implement against.

## Decision

The bc-portal frontend is React 18.3.1 + TypeScript 5.6.3 + Vite 6.3.5 with TanStack Query, Radix UI, and Tailwind CSS. The original plan incorrectly referenced Angular. All backend API design must target the existing React hooks and api/client.ts fetch wrapper.
