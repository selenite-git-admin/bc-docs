# Contract Chain Reconciliation — 3 active chains + AI provisional, 4 execution boundaries

## Context

The architecture evolved through D022→D044→D050→D054→D069→DEC-0e3c64 over 24 days. Each decision refined the model but nobody updated the essentials docs. This reconciliation locks the final state and mandates updating all stale references.

## Context

Three competing models existed in authoritative documentation:

1. **D022 (DEC-ac05fc, Mar 1, SUPERSEDED):** 3 chains — Source (extraction→source→admission→canonical), Metric (metric→intervention), AI (ai). Canonical was INSIDE Source Chain.
2. **DEC-0e3c64 (Mar 24):** 3 chains — Source (SC→AC), Canonical (CC + Mapping Binding), Metric (MC→IC). Canonical became its OWN chain. AI not mentioned.
3. **Foundation spec (LOCKED):** No chains concept. 8 boundaries. 6 contract types. 4 primary objects.

Additionally, evaluation-boundaries.md (essentials) said "exactly 4 boundaries" but listed 5.

## Decision

### Chain Model (3 active + 1 provisional)

| Chain | Contract Types | Purpose | Status |
|-------|---------------|---------|--------|
| **Source Chain** | SC → AC | Plumbing — declares fields, governs admission | Active |
| **Canonical Chain** | CC + Mapping Binding (OC + CM) | Meaning — resolves source semantics into business vocabulary | Active |
| **Metric Chain** | MC → IC | Measurement — computes KPIs, governs intervention | Active |
| **AI Chain** | AI | Intelligence — interpretation and anticipation | **Provisional** (tables exist, no runtime) |

- Canonical is a chain, not a bridge (despite originating as D050's "bridge" concept)
- OC belongs to Canonical Chain (per DEC-0e3c64), not Source Chain
- Mapping Binding = OC + observation_field_map + canonical_mapping (first-class Canonical Chain entity)
- AI Chain is provisioned in the database (ai_contract tables exist) but has no runtime implementation yet

### Boundary Model (4 execution boundaries)

| # | Boundary | Input | Output | Contract |
|---|----------|-------|--------|----------|
| 1 | Admission | Observed Records | Source Object (SO) | Admission Contract |
| 2 | Canonical Evaluation | N Source Objects | Canonical Object (CO) | Canonical Contract |
| 3 | Metric Evaluation | N Canonical Objects | Metric Snapshot | Metric Contract |
| 4 | Intervention Evaluation | Metric Snapshots | Action Object (AO) | Intervention Contract |

The Foundation's Observation Boundary exists but is NOT one of the 4 execution boundaries — it precedes the chain model.
