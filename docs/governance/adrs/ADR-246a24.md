# Stack naming: bc-{stage}-{series}-{domain}

## Context

Monolithic single stack was unwieldy — deploying auth also deployed VPC+Aurora (expensive, unnecessary). Separate stacks with numbered series give clear dependency ordering, independent lifecycle management, and room for growth without disrupting existing stack names.

## Decision

CDK stacks use naming convention bc-{stage}-{series}-{domain} with 4-digit series numbers spaced by 100s: 0100-network, 0200-auth, 0300-data, 0400-compute. Series spacing allows inserting stacks without renumbering. Numbers represent logical dependency order, not deployment order (independent stacks can deploy in any order).
