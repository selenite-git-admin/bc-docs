# Navigation IA v4 — Source Catalog, Source Chain, Canonical Map, Metric Chain

## Context

Superseded by D068. Source Contracts removed from nav (contracts are Reader-scoped, surface from Reader detail page). Source Objects removed from nav (Reader output, surfaces from Reader detail page). Source Catalog reduced to 2 items, Source Chain reduced to 2 items.

## Decision

Supersedes D060. Four groups restructured:

**Source Catalog** (passive inventory, 3 items):
1. Source Landscape — per-source reference indexes (SAP tables, CDS views)
2. Source Catalog — all providers in single page with category filter chips (Enterprise/Third Party/Public/Synthetic). Replaces 4 separate nav items.
3. Source Contracts — extraction + source + admission contracts (surfaced in nav, was hidden)

**Source Chain** (active execution, 3 items):
1. Connectors — adapter registry
2. Readers — UinBAT reader definitions
3. Source Objects — SO list (moved from Source Catalog, belongs here as reader output)

**Canonical Map** (wiring layer, CO at center, 3 items):
1. Canonical Objects — CO definitions (moved from Metric Chain)
2. Field Bindings — SO→CO field mapping (moved from Source Chain, renamed from Field Resolution)
3. Metric Bindings — CO→MO metric binding (moved from Metric Chain, renamed from MO-CO Mapping)

**Metric Chain** (metric-specific, 2 items):
1. Metric References — reference data
2. Metric Objects — MO definitions

CO Map visualization (SO←CO→MO) lives on CO detail page as a section, not a standalone nav page.

Key changes: 4 source category nav items collapsed to 1 page with filter chips. Source Objects moved Catalog→Chain. CO and mappings extracted into new Canonical Map group. Consistent "Bindings" naming (Field Bindings, Metric Bindings).
