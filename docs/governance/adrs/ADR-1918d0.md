# Deployment & Database Architecture

## Context

DB audit showed 59 tables, 730+ columns grown decision-by-decision without a unified design pass. bc-admin audit confirmed all tables are consumed but many columns are theoretical. Shared services from D008 (17 originally planned) needed triage — some done, some AWS-managed, some deferred. No deployment architecture was documented.

## Decision

### Staging Topology (~$64/month)

- **bc-core:** ECS Fargate
- **bc-portal, bc-admin:** S3 + CloudFront (static SPA)
- **Database:** RDS PostgreSQL
- **Auth:** Cognito (existing)
- **API Gateway:** AWS API Gateway
- **IaC:** CDK
- **CI/CD:** GitHub Actions
- **No Redis** — not needed day 1
- **Domain:** *.barecount.com
- **Single staging env** (no dev/staging split)

### Database Separation

- **Platform DB** (`bc_platform_dev`): 82 tables across 12 schemas — registry, contract, source, runtime, operations, master, metric, tenant, infrastructure, support, pricing, users
- **Tenant DB** (`tbc_{slug}_dev`): 27 tables per tenant across 3 schemas — boundary data, evidence, tenant-scoped config

### 10 Mandatory Normalization Rules

1. No JSONB for queryable data
2. No denormalized counters
3. FK constraints mandatory
4. One source of truth per value
5. Shared patterns use shared tables
6. Max 20 columns per table
7. Indexes follow query patterns
8. Soft deletes: `archived_at`
9. Temporal: `effective_from` / `effective_to`
10. New table checklist before adding any table

### Shared Services Triage (D008 superseded)

- 7 done (auth, tenancy, config, schema registry, audit, governance, policy)
- 5 AWS-managed (secrets, observability, storage, gateway, scheduling)
- 5 build-later (data protection, compliance engine, notification, integration bus, billing)

## Options Considered

| Option | Pros | Cons |
|--------|------|------|
| **ECS Fargate + RDS (chosen)** | Simple, cost-effective, scales to production | No container orchestration flexibility |
| EKS (Kubernetes) | Industry standard for microservices | Overkill for single API service, $73/mo for control plane alone |
| Lambda + Aurora Serverless | Pay-per-request, scales to zero | Cold starts, connection limits, harder debugging |

## Consequences

1. **Full spec maintained** at `architecture/D089-deployment-database-architecture.md`
2. **Normalization rules enforced** per-session — added to CLAUDE.md
3. **bc-sdg is dev-only** — not deployed to staging
4. **Port 5435** for PostgreSQL (Docker redesign)
5. **Schema-map.md** is the authoritative table inventory
