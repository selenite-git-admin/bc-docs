# Registry tables aligned with UinBAT Reader terminology

## Context

Post provisional patent, the platform adopted UinBAT Readers as the semantic ingress component operating at the Admission Boundary (Component Ref 090). The old generic consumer/consumption terminology did not reflect that schemas are admission contracts, consumers are domain-specific readers, runs are admission executions with observe/admit/reject semantics. Aligning table structure to authoritative component references ensures the schema registry accurately models the platform's boundary architecture.

## Decision

Replaced generic consumer/schemaConsumption/schemaRun tables with reader/readerBinding/admissionRun tables aligned with UinBAT Reader (Universal Business-Aware Transaction Reader) architecture. Reader table captures domain (AR, AP, GL) and source system. ReaderBinding links readers to admission contracts (schema versions) per source entity. AdmissionRun tracks reader execution with records_observed/records_admitted/records_rejected counters. TenantBinding simplified (dropped effectiveVersion, override). SchemaCategory expanded with admission and canonical types.
