# Editorial Amendment of Active Characteristic Definitions — Refinement of DEC-26b6e2

## Context

No rationale recorded.

## Decision

Refines DEC-26b6e2 (Immutable Characteristic Atoms) — does NOT supersede it. DEC-26b6e2's core stands: active characteristics are immutable semantic atoms, the term is frozen at publication, and supersession (minting a new characteristic_id) is the principal correction path.

This refinement formalizes the operational meaning of "editorial" for the in-place amendCharacteristicDefinition endpoint (PR #343 / TSK-4c6fbd), which edits a characteristic's definition in place under a (registry_amend_definition, characteristic) cert and an append-only characteristic_definition_amendment ledger row, gated by the DB CHECK correction_class='editorial'.

DECISION: The in-place amend-definition endpoint is the doctrinally-correct path IF AND ONLY IF all six conditions E1–E6 hold. If ANY condition fails, the change is meaning-bearing and MUST go via supersedeCharacteristic (new characteristic_id, with characteristic_supersession.correction_class carrying the audit classification).

E1 — The characteristic term is unchanged (mechanically enforced; the amend endpoint never touches characteristic.term).
E2 — The bound representation_term, data_type, and (D442) semantic_role are unchanged.
E3 — No new semantic/denotation class is introduced (the kind of thing the characteristic refers to is the same before and after; removing entity/source scope-wording for the SAME denotation passes; adding a genuinely different class fails).
E4 — The amendment clarifies/removes accidental wording, OR aligns the definition with already-governed active bindings (editorial purpose is to make the definition true to the substrate already in place, not to expand into new substrate).
E5 — No existing valid binding becomes invalid (broadening never invalidates a narrower binding; a narrowing that contradicts an active binding fails).
E6 — No future binding becomes valid SOLELY by semantic expansion into a new class (new bindings may become valid only if their class was already represented in the substrate per E4).

Operational interpretation of "new class" (E3/E6): a class is "already represented" iff at least one active BC on the characteristic at amendment time binds an entity of that class; the class taxonomy is the §11.6 operational one (invoice/order/payment/receipt/shipment/etc.). Default when uncertain: supersession (editorial is the narrow path, supersession the safe path).

This ratifies the precedent of the 2026-06-23 net amount + gross amount amendments (provenance-leak removal aligning with already-governed bindings = editorial) and the negative precedent of the 2026-06-20 delivery date broadening (introduced a genuinely new denotation class — document receipt — and was correctly reversed; that shape MUST use supersession).

CONSEQUENCES: Editorial amendments preserve characteristic_id + term and produce one characteristic_definition_amendment ledger row. Meaning-bearing changes mint a new characteristic_id; predecessor BCs remain historically correct; rebind is a separate governed act. The six-condition test makes the editorial/meaning-bearing classification deterministic at amendment time. DEC-26b6e2's no-silent-rebind guarantee is preserved (E5).</decision_text>
