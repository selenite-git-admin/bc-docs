# Source system tabs by business category, not transport mechanism

## Context

Foundation Ref 090 (UinBAT Reader): source system identity and extraction method are separate concepts. Admission Contracts bind to source system identity first. Current tab structure teaches users the wrong mental model by elevating transport mechanisms (S3, Snowflake) to source system status. Categorizing by business domain (ERP, CRM) aligns with how users think about their enterprise landscape.

## Decision

Replace the 3-tab structure (Operational Systems / Data Stores / Files & Storage) with tabs by business system category (ERP, CRM, HRMS, ITSM, Marketing, Public Data). Snowflake/S3/OneDrive become admission methods on source system cards, not standalone entries. Add Public Data tab with real ECB/WB/FRED connectors from valustream-data-acquisition repo.
