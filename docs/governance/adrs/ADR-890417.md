# pm2 for multi-project dev service management

## Context

Manual per-session startup was error-prone and slow. Docker Compose considered but rejected due to Windows/OneDrive file system performance penalty for Node projects. pm2 gives one-command startup, individual restarts, log aggregation, and process monitoring without the container overhead. Port assignments follow D014.

## Decision

Use pm2 ecosystem.config.cjs in DevHub to start/stop/restart all BareCount dev services from one place. Services: bc-core (3100), bc-portal (3000), bc-admin (3010), intra-site (3020), devhub (4000), bc-core-dashboard (4100). Commands: npm run dev:up, dev:down, dev:restart, dev:logs, dev:status, dev:mon. Individual restart via npx pm2 restart <name>. For Windows npm.cmd compatibility, bc-core uses scripts/service-runner.js wrapper. All Vite services use --host for dual-stack IPv4/IPv6 binding. .claude/launch.json mirrors the same services for preview_* tool integration.
