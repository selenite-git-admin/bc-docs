# Housekeeping Agents Migration — bc-ai APScheduler → Claude Code Scheduled Tasks

## Context

1. Cost: bc-ai agents called Bedrock Haiku for classification/reasoning — Claude Code IS the reasoning engine, no extra LLM cost. 2. Reliability: MkDocs agent was broken due to hardcoded paths + bc-ai dependency. Claude Code has direct filesystem access. 3. Capability: bc-ai agents could only recommend actions via HTTP proxy chain (DevHub UI → bc-ai → DevHub API). Claude Code agents have native MCP access to all devhub_* tools. 4. Simplicity: removes APScheduler dependency, Python async complexity, and HTTP proxy layer. 5. Landscape awareness: Claude Code agents run in the same environment as development, with access to git, all repos, and full codebase context.

## Decision

Migrate all 8 housekeeping agents from bc-ai (Python/APScheduler/Bedrock Haiku) to Claude Code scheduled tasks running locally. Disable the APScheduler in bc-ai. The bc-ai housekeeping HTTP endpoints remain available for manual triggers but no longer run on a schedule.

Agents migrated: registry-refresh, doc-staleness, session-hygiene, task-triage, nc-aging, risk-review, qa-patrol, mkdocs-maintainer.

Schedule: daily agents staggered 00:02–00:52 UTC (~5:30–6:22 IST), registry-refresh every 6h, qa-patrol weekly Saturday.
