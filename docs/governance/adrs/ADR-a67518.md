# Tenant Onboarding Gate — checklist-based readiness for BYO-DB and BC-Agent tiers

## Context

BYO-DB and BC-Agent tiers require real-world pre-work (infrastructure provisioning, hardware purchase, network setup) that must complete before the platform can create a tenant. Without a gate, tenant creation would fail or waste engineering effort troubleshooting customer infrastructure. The onboarding record makes the prerequisite explicit and trackable.

## Decision

BYO-DB and BC-Agent hosting tiers require a completed Onboarding Record (ONB-xxxxxx) before tenant creation is allowed. Onboarding records are created by sales during customer engagement and track a tier-specific checklist (sales agreement, infrastructure readiness, connectivity verification, technical sign-off). Status lifecycle: initiated → in_progress → ready → activated → cancelled. Tenant creation for these tiers is gated — the system requires a linked onboarding record in 'ready' status. AWS Shared and AWS Separate tiers are self-service with no onboarding gate. bc-admin gets an Onboarding management page under the Subscription menu group.
