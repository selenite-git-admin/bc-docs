# Email OTP MFA deferred — TOTP retained with UX improvements

## Context

Cost-benefit analysis: TOTP is free on Lite tier, more secure (phishing-resistant), and already wired end-to-end. Email OTP adds $10+/mo minimum, forces phone number collection, breaks current SDK choice, and requires SES DEVELOPER mode setup. For a B2B platform with admin-provisioned users, guided TOTP onboarding is sufficient. The UX gap can be closed with good setup flow rather than switching MFA method.

## Decision

Keep TOTP-only MFA (D014 unchanged). Email OTP MFA deferred due to: (1) Requires Cognito Essentials tier at 2.7x cost ($0.015 vs $0.0055/MAU), (2) Recovery channel conflict — email MFA forces phone-based recovery, requiring phone number collection + SNS SMS setup, (3) amazon-cognito-identity-js SDK doesn't support EMAIL_OTP challenge type — would require SDK rewrite to @aws-sdk/client-cognito-identity-provider or Amplify v6. Instead: build guided MFA setup UX (TSK-0f2ab4) with authenticator app download links + QR scan. MFA stays OPTIONAL, admin-enforceable per user. Revisit email MFA when Essentials tier is justified by scale, SES is already configured for transactional emails, and phone collection is natural via profile completion.
