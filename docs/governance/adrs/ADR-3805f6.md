# Screen Registry workflow taxonomy: split dataspace into canonical-data, evidence, data-infra

## Context

bc-portal route restructure created distinct /canonical-data/*, /evidence/*, /data-infra/* namespaces. Registry workflows should mirror route namespaces for screen-by-screen data seeding in Phase B.

## Decision

Replaced the monolithic 'dataspace' workflow with targeted workflows: canonical-data (19 screens for contracts, schemas, categories), evidence (2 screens for explorer + lineage), data-infra (9 screens for connections, retention, compliance). Source-data (13 screens) remains for ingestion-side routes.
