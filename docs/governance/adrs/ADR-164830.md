# DB Change Protocol — mandatory user approval for all schema changes, overrides Bypass permissions

## Context

Database is the foundation — changes cascade through APIs, services, and UI. The blast radius of a schema change is the largest of any code change. User (founder/architect) must review every schema decision before execution to prevent cascading rework.

## Decision

No database schema change may be made without explicit user approval. This applies even in "Bypass permissions" mode. Covers: adding/removing/renaming tables or columns, changing types/constraints/defaults/indexes, Drizzle schema definitions, migration files, DDL files, seed data structure. Protocol: (1) present change with table name + what + why, (2) wait for explicit approval, (3) execute. Added to CLAUDE.md and memory.
