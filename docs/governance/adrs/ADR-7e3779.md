# Source Catalog Dual-System Strategy — ECC Primary, S/4HANA Overlay

## Context

SAP S/4HANA core FI tables are backward-compatible with ECC — BSID, BKPF, BSEG have identical field structures in both systems. Storing them twice wastes space and creates sync issues. The sapdatasheet.org cache (110K+ tables, 796MB) covers both systems. ECC-first with S4 overlay eliminates duplication while preserving system-specific tables. bc-seed MongoDB replaces fragile S3 tar.gz archives with a queryable, versioned, live-scrapeable service.

## Decision

SAP source catalog uses ECC as the primary system with S/4HANA as an overlay. Shared tables (BSID, BKPF, BSEG, KNA1 etc.) are stored once under system=ecc and serve as fallback for S/4HANA lookups. S/4HANA-only tables (ACDOCA, MATDOC, I_JournalEntry) are stored separately under system=s4hana. sapdatasheet.org is the authoritative source for field-level metadata (name, type, length, key, description, domain, data element). Data resides in bc-seed MongoDB, not S3 archives.
