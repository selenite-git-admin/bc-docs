# Source Catalog shows system cards, not provider cards

## Context

Users think in systems they use ("I use S/4HANA") not vendors ("I use SAP SE"). Provider-level cards forced all 8 SAP systems into one card, making the ERP/CRM/HCM type filter meaningless. System-level cards correctly distribute SAP's products across type filters.

## Decision

The Source Catalog page in bc-admin displays source_system cards (S/4HANA, Salesforce Cloud, ServiceNow ITSM) not source_provider cards (SAP SE, Salesforce Inc). Provider is a backend/engineering concept with no practical use case in admin or portal apps. Each card shows system name, provider name (subtitle), provider logo, system_type badge, status badge. Filtered by system_type chips (dynamic per category) and category chips (Enterprise/Third Party/Public).
