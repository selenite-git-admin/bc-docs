# bc-website v2 Direction — Marketing-First, Separate Voice Register

## Context

v1 is a specification wearing a nav bar — brilliant writing but terrible for customer acquisition. A CFO reading "meaning is resolved at the admission boundary" closes the tab. The website must sell outcomes, not teach architecture. v1 content retains value as blog/SEO/whitepaper content for prospects already in conversation.

## Decision

v2 website uses business outcome language (consequences, accountability, finality) — NOT system language (admission boundary, canonical evaluation). The locked language system (foundation/governance/) governs system documentation. The website gets a separate marketing voice register. When referencing system internals, link to blog/docs where locked vocabulary applies. v1 content (12 pages of architectural prose) becomes blog posts and whitepapers, not the homepage.
