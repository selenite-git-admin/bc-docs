## Decision — A1 Target-state catalogue + admission rule

- sec1 Evidence-Governed Foundation Buildout adopted as the load-bearing admission rule for the program scope.
- AR-3 entity_backbone_completeness + AR-4 standards_backed_foundation declared sufficient by construction. AR-1 known_metric_pull, AR-2 source_chain_pull, AR-5 operator_strategic_priority are optional strengthening evidence, not required.
- sec4A target catalogue accepted: 54 proposed new entities, 217 proposed new characteristic bf_names, 733 BC targets.
- sec4A.10 admission rationale distribution accepted.
- sec4B layer-first re-aggregation accepted.
- sec6.3 row schema accepted (admission_rationale, used_by_bc_target_count, entity_slice_name, composite_identity_decision, packet_hash, retry_reason, 15 parallel-execution fields including coordinator_run_id, worker_id, claim_expires_at, substrate_snapshot_hash, evidence_hash, model_cost_estimate, model_cost_actual, dependency_group, write_queue_position, final_prewrite_check_at, final_prewrite_check_result, terminal_outcome_code).
- A0.5 per-characteristic shape matrix in bcf-oagis-a0.5-template-catalogue-2026-06-24.md sec1.C1-sec1.C6 is the Cat-1 pin authority for 217 new characteristic admissions. Cluster groupings are review-organization, not approval authority. Cat-2 entity-slice entity list in catalogue sec5 is the Cat-2 pin authority for 54 entity admissions. Cat-3 OAGIS T2 citation is format only — per-row provenance + evidence_hash remain row-level.
- Safeguard S1 (used_by_bc_target_count >= 1) and S2 (entity_slice_name non-null) enforced.

## Decision — A2 Row classification + admission rules

- sec5.1 row execution classification (GREEN / AMBER / RED / DEFER / UNKNOWN) accepted.
- sec8.2 row execution rules accepted: GREEN executes autonomously; AMBER executes autonomously only when row carries pin_evidence_text from the A0.5 matrix; RED never executes autonomously (separate operator packet); DEFER never executes (held until pull condition changes); UNKNOWN fatal stop.
- sec1.2 admission rationale validation: each row carries >=1 AR code with evidence (default AR-3+AR-4 per sec20.1 posture).
- sec1.4 safeguards S1 + S2 enforced at A0 compile time and at writer pre-write recheck time.

## Decision — A3 Halt rules

- sec5.3 per-(C/E/BC)-wave halt rules accepted.
- sec8.4 22 fatal stop triggers accepted (service unavailable, auth fail, schema/prompt/parser fail, repeated park reason, malformed evidence, write outside F3/C5, new char/entity in reuse row, representation_term ambiguity not pinned, budget/time cap reached, Foundation invariant violation, DEC-fb0b12 ambiguity without pin, system-agnosticism failure, resolver-stamped attempt, forbidden term, source-field copy, cross-registry mutation, telemetry missing, direct-write detected, prompt/cache drift, composite-identity acyclicity failure, UNKNOWN row encountered).
- sec6A.0/sec6A.1 compile-layer halt pre-conditions accepted.

## Decision — A4 Retry ledger + Compile + Cache + Coordinator architecture

- sec6 retry ledger (Phase-1 Markdown at bcf-oagis-retry-ledger-2026-06-24.md) accepted as substrate-of-record.
- sec6A A0 Compile / Preflight Layer accepted as the only approved dry-run style (read-only, no LLM, no panel, no write).
- sec6B Batch + Cache Execution Economy accepted (six cache layers, invalidation triggers, retry economy; future microbatch panel design NOT authorized).
- sec6C Coordinator / Workers / Single Writer architecture accepted with concurrency defaults: A0 read/classification workers 4-8, evidence packet workers 4, panel workers 2 concurrent (ramps to 3 after 20 consecutive clean panel outcomes), writer cardinality 1 (serialized single lane), FIFO within dependency_group.
- sec6.7 program-completion state header accepted.

## Decision — A5 Budget / time caps

- Total program spend cap: $400 USD (cumulative model_cost_actual ceiling).
- Total panel-call cap: 1,216 panel invocations (973 executable + 25% retry buffer).
- Optional pass-level caps: Pass 1 (C-waves) $80 / 270 panel calls; Pass 2 (E-waves) $40 / 80 panel calls; Pass 3 (BC-waves) $280 / 880 panel calls.
- Per-row latency cap: 180 seconds (work_claimed_at to final_prewrite_check_at).
- Wall-time cap: 24 hours (program-start to now).

All caps tracked cumulatively by coordinator; breach fires sec8.4 fatal stop trigger.

## Decision — Open Question resolutions

- OQ-1 AMBER pin posture: accept per-row pin_evidence_text from A0.5 catalogue matrix (do not wait for packet-builder enhancement).
- OQ-13 RED row sequencing: sequential one packet at a time.
- OQ-14 sequencing model: layer-first per sec1A.
- OQ-15 first-wave sequencing: sequential C1 -> E1 -> BC-AR for first program execution.
- OQ-16 admission rationale review depth: distribution-level operator approval at A1 + per-row catalogue (matrix captures per-row pins).

## Decision — 7 RED composite-identity packets queued for separate operator decisions

Inventory Position; Work Order Operation; Operation; Bill of Materials Line; Inspection Lot; Test Result; Maintenance Order Operation. Each requires its own operator decision packet per sec1.4 + sec5.1 + BCR sec4 acyclicity confirmation.

## Decision — Known deferred item

Existing characteristic 'effective date' has multi-shape OAGIS proposals (date/date/temporal vs date/timestamp/temporal). Matches the known Wave A Customer Invoice case from scope audit sec3.5. Surfaces as per-row pin at execution time, not at A1.

## What this recording does NOT do

- Does NOT start panel execution. Separate operator act required to begin Pass 1 C1.
- Does NOT touch concept_registry.* or bcf.* substrate. Substrate confirmed unchanged at recording: 26 active entities / 62 active characteristics / 194 active value BCs.
- Does NOT authorize RED row execution. RED rows remain held; separate operator packet per entity required.
- Does NOT lock the operator out of revising any cap or decision. Operator may amend by reply or by superseding this ADR.
