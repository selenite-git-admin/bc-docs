# AI-assisted catalog verification — hybrid model with gated enterprise controls

## Context

Two goals: (1) Accelerate source onboarding — 63 providers × thousands of objects can't be manually verified at scale. AI pre-verification handles 90%, human confirms. (2) Enterprise demo — buyers need to see AI is gated and auditable, not autonomous. This workflow is the proof point.

## Decision

Source catalog verification uses a hybrid AI model:

1. **Pre-computed at seed time**: Enrichment scripts call Claude API in bulk to generate draft verification notes per module/object/field. Stored in DB as structured JSON (verification_notes).

2. **UI review**: Platform admin sees pre-computed AI notes. Can approve immediately if confident, or click "Deep Verify" for a live AI call with richer context.

3. **Controls**: Token budget per batch. Locked prompt (not user-editable). Structured JSON output with confidence score. Every AI output stored as evidence with prompt hash, model version, timestamp. Human always has final decision — AI recommendation ≠ auto-approval.

4. **Tier split**: Provider/System/Version auto-approved by seed (identity facts). Module/Object/Field stay 'registered' until human verifies in UI with AI assistance.

5. **API shape**: `POST /api/source-catalog/verify` — accepts entity tier + ID, returns structured verification result. CatalogVerificationService calls Claude API with scoped prompt.

6. **Commercial value**: Demonstrates gated enterprise AI — scoped, controlled, auditable, human-in-the-loop. Key differentiator for enterprise trust.
