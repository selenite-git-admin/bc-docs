# Domain Taxonomy as Platform Master

## Context

Three separate TypeScript files define domain names today with manual sync between bc-core and bc-admin. Platform Masters gives single API-driven SSOT, eliminates hand-copied duplicates, and allows future tenant-level domain customization.

## Decision

The 19-domain business taxonomy becomes a Platform Master — a platform-level master data table served via API. Current hardcoded sources (metric-taxonomy.ts METRIC_DOMAIN_LABELS, generate-catalog-contracts.ts CO_PREFIX_TO_DOMAIN) become seed data for the master table. All consumers (metric-catalog, business-catalog, bc-admin, bc-portal) read from the API instead of local copies. The old 29-domain domain-taxonomy.ts (Taxonomy B) is deprecated — its granular slugs are subsumed as subdomains under the 19 coarser domains.
