# Screen Registry: design_rationale captures scope — no separate scope columns

## Context

Scope is design intent, not registry metadata. Separate columns would create 3 overlapping freetext fields (notes, design_rationale, scope). The 'Purpose / Not here' convention keeps scope embedded in rationale without schema bloat. Formal scope governance lives in DevHub decisions and plans.

## Decision

Use the design_rationale column with a 'Purpose / Not here' convention to capture both what a screen does and what's explicitly excluded. No dedicated scope or out_of_scope columns.
