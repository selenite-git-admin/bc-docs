# Operational config and connections are tenant-scoped, not on reader definition

## Context

The reader is a platform-level capability template. The tenant decides scheduling, retry policies, and provides credentials. Same reader, different operational configs per tenant. Showing operational config at the reader level is misleading.

## Decision

Reader definitions (bc-admin, platform scope) do NOT include operational config (schedule, retry, backfill, failure policy) or connections (endpoint, credentials). These are tenant-scoped — each tenant decides when/how to run the reader and provides their own connection instances. Reader detail page shows only: identity, target sources, connectors (adapter type), contracts, and health signals.
