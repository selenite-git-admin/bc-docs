# V2 Seed Engine — dedicated, complete, AWS-ready

## Context

1. Schema restructured (god-table split) — old data doesn't map cleanly. 2. AWS deployment needs reproducible provisioning from zero. 3. Deterministic, version-controlled seed data vs. DB dump. 4. Clean testing — fresh instance every time. 5. User directed: no fragmented wave-by-wave data, complete profile in v2.

## Decision

V2 database uses a dedicated seed engine (not data transfer from v1). One command (`npm run seed:v2`) provisions a complete instance from zero. All data seeded in one pass: 2,428 KPIs (818 approved + 1,610 registered with status), all 63 enriched source providers, 244K SAP tables, 415 modules, all standard fields, all master data. No NestJS dependency — direct Drizzle + node-postgres for speed and CI/CD compatibility. Same engine runs locally and in AWS staging/prod. Naming convention: v2 (not "redesign"). DDL folder: docker/v2/, seed folder: seed-v2/.
