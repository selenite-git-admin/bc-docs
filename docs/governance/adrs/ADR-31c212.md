# MCF metric function/subfunction classification: master-authoritative, not_defined sentinel, DB-enforced

## Context

The prior design let unvalidated free-text classification enter at intake and then dropped it at materialization, and the seed catalog's own classifications are uncontrolled free text (197/262 subfunctions absent from master). Preserving the seed assertion would propagate noise. Making master the authority + a not_defined sentinel + DB-level FK/NOT NULL guarantees every metric carries a master-valid classification, with the panel doing the semantic placement into the closed vocabulary. DB enforcement (vs service validation) prevents recurrence; the metric clean-slate makes NOT NULL free to add now.

## Decision

**Problem.** MCF metrics were structurally incapable of carrying a function classification. (A) the M11 operator-direct intake admits `function_code`/`subfunction_code` as unvalidated free text inside an opaque `@IsObject()` `normalized_candidate_json`; (B) the panel's `CandidateProposal` interface has NO function/subfunction field — the panel never classifies; (C) `MetricAuthoringMaterializationService.composeMetricContractVersionInput` hardcodes `functionCode: null, subfunctionCode: null`. Net: every MCF-authored MCV lands `function=null`, invisible to every function-scoped surface (readiness funnel, catalog, from-seed scoping). The seed catalog is not a trustworthy source either: `mcf.seed_metric` carries free-text classifications — 262 distinct subfunction values, only 65 in `master_subfunction`, 197 uncontrolled (master holds 99).

**Decisions.**
1. **Master is the complete authority.** `master_function` (19) + `master_subfunction` (99) are the controlled vocabulary. The seed's 197 extra subfunctions are noise — discarded, NOT reconciled into master. Master grows only by deliberate platform governance, never by importing seed free text.
2. **`not_defined` sentinel** added to `master_function` (1 row) + `master_subfunction` (one per parent function, composite-PK clean) — the explicit fallback when no master slug fits. Authoring never blocks; the unclassified set stays queryable.
3. **DB-enforced, not service-validated.** `mcf.metric_contract_version.function_code`/`subfunction_code` become `NOT NULL DEFAULT 'not_defined'` + a composite FK `(function_code, subfunction_code) → master_subfunction(parent_function, slug)`. Null and non-master classification become structurally impossible. Safe to add now: the metric clean-slate (SES-f3b73c, 2026-06-27) left 0 MCV rows.
4. **The platform panel classifies.** `CandidateProposal` gains `function_slug`/`subfunction_slug`; the Maker classifies into the closed master list (choosing `not_defined` when nothing fits); the Checker validates the pick is a real master slug; materialization writes the validated slugs to the MCV. Classification is a platform property (MC is platform-only, D164) — never tenant-scoped. The tenant `t/admin/functions` console (D346) manages tenant bindings only and is the wrong home for vocabulary or classification.
5. **Deferred re-assignment is clean.** function/subfunction are descriptive MCV metadata — NOT in the identity tuple or package signature — so a metric parked at `not_defined` is re-assignable later, even once `active`, without tripping the Invariant-III immutability guards. "Add a master entry, then assign the correct value" works.
6. **Delivered via `docker/redesign` migration, not CSV seed.** A value the validation gate structurally depends on must be guaranteed; CSV seeds are load-order/environment dependent. The migration (idempotent) is authoritative for the sentinel; CSVs may mirror for fresh-seed convenience only.

**Foundation / repair-location.** Root is **B** (the authoring contract carried no function field); the fix spans B (CandidateProposal) + A (intake validation) + C (materialization binding) + E (storage constraint). An intake-only validation patch would be A-level compensation for a B gap — forbidden by the gate.

**Consequences.** The materialization's hardcoded `null` now fails loudly (FK/NOT NULL) until B/C land — intended. ARPI lands `finance`/`billing` (both already master-valid). Follow-ups: a governed `not_defined` re-assignment worklist; the seed's 197 free-text subfunctions are NOT triaged into master.
