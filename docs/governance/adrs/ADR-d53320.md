# Three SAP landscapes: S/4HANA Public Cloud, S/4HANA On-Premise, ECC EHP8

## Context

Proves BareCount handles the same business processes from different SAP versions — the strongest demo point for prospects. Real clients run mixed SAP estates. Showing convergence from 3 landscapes to 1 canonical form demonstrates the platform's core value proposition.

## Decision

bc-sdg simulates three distinct SAP landscapes with genuinely different data shapes: (1) S/4HANA Public Cloud — OData V4, CDS views (I_JournalEntry etc.), ACDOCA Universal Journal, Business Partner (BUT000), restricted extensibility. (2) S/4HANA On-Premise — OData V2/V4, CDS views + classic table access, ACDOCA + BSEG compatibility views, BP + KNA1/LFA1 dual model, full ABAP extensibility. (3) ECC EHP8 — RFC/BAPI-style payloads, classic tables only (BKPF/BSEG/BSID/BSIK/KNA1/LFA1), IDocs, New GL optional. All three converge to identical BareCount canonical objects, proving source-agnostic evaluation.
