## 2. The bug

`ContractRepository.createContract` (the SQL writer behind every
`McOnboardingService.create()` call) omitted **`metric_definition_id`**
from its INSERT for the `metric` family. The column is `NOT NULL` on
`contract.metric_contract` with no default — INSERT failed at runtime
with a NOT NULL violation. `category_code` was a secondary concern:
the column has `DEFAULT 'metric'::text`, so its absence from the INSERT
is harmless.

The 778 historical MCs in the catalog must have been created via paths
other than `McOnboardingService.create()` end-to-end — likely seed-time
direct inserts, wizards, or a different service since removed. This
code path had never actually written a `metric_contract` row.

A secondary bug surfaced in `SeedPromotionService`:

- It didn't pass `name` in the `CreateMcDto`, so the service fell back
  to `mc__o1` (slugified formula LHS), not `mc__revenue_collection_rate`.
- It hardcoded `iteration_seq: 1`, which would collide with the
  `UNIQUE (metric_formula_id, iteration_seq)` constraint on retry.
