## 2. The bugs

### 2.1 IntegrityService SQL typo

`integrity.service.ts:1283` was querying `status` against `contract.business_field`, where the live column is `status_code`. JS-side consumer reads `row.status`. Patch: `status_code AS "status"` keeps consumer signature stable. **Transitional fix only** — IntegrityService is deprecated by ADR-bebaec / D305; the patch exists because `ContractService.transitionState:456` still calls `getKpiIntegrity` as the pre-activation gate.

### 2.2 MLS-14 vs ChainStatusService sequencing contradiction

MLS-14 reads `contract.chain_status.chain_verdict` via LEFT JOIN as a pre-activation gate. For a fresh MC, no chain_status row exists yet → verdict is NULL → refusal with `MLS-14.chain_not_complete` (BLOCKER). The 540 historical MCs predate D391 (May 4 2026) and never traversed this combined gate.

Resolution: new method `ChainStatusService.refreshChainStatusForVersion(metricContractId, versionCode)` computes chain status for any version regardless of `governance_state_code`. Called synchronously by `ContractService.transitionState` between IntegrityService (line 456) and MLS-14 (line 479). Upstream CCs/OCs remain active-state-filtered.

### 2.3 MetricWizardService.completeMetric — parallel writer

`metric-wizard.service.ts:219` wrote `contract.metric_contract` + `contract.metric_contract_version` directly via raw INSERT, bypassing every gate. **Quarantined at both controller and service layers** (defense in depth). The 250+ lines of legacy code are preserved unchanged below the `throw` for reference. Canonical replacement: `POST /api/onboarding/seed/:metric_definition_id/promote`.

## 7. Hard rules honored

- IntegrityService patched only as a transitional fix; comment explicitly names it as deprecated.
- `chain_not_complete` BLOCKER is **not** skipped or weakened. The fix removes the false-negative cause (missing chain_status row for fresh MCs); MLS-14's evaluation logic is untouched.
- Missing chain_status is **not** treated as pass. Pre-activation compute either produces a real verdict (`complete | partial | broken | unlinked`) or aborts activation with `ForbiddenException`.
- MetricWizardService.completeMetric blocked at two layers; legacy code preserved unchanged.
- Wizard's other endpoints (session management) untouched.
- TestBench, Inspector, McIntegrity untouched.
- Draft MC, draft version, deterministic verification rows from prior slice (1) attempts all preserved.
