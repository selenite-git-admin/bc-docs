# v4 Documentation Areas

Documents land here only after migration decisions are recorded in the control database.

The current navigation is generated at [NAVIGATION.md](NAVIGATION.md). It mirrors the reader flow while keeping governance and evidence records outside the primary path.

Reader-flow areas:

- `overview/`
- `foundation/`
- `operating-model/`
- `implementation/`
- `ai/`
- `development/`
- `onboarding/`
- `operations/`
- `compliance/`

Supporting areas:

- `reference/`: generated and curated lookup material.
- `governance/`: ADRs, errata, and policy records.
- `evidence/`: DBCPs, closeouts, ledgers, work records, and audit evidence.
- `archive/`: retained historical material that is not current reader truth.

Generated API, schema, and data-dictionary references are intentionally absent until they can be rebuilt from current source.
